Embed the NEON RUSH slot

A neon slot-machine game you drop into any web page with two lines. It's a standard Web Component — works in plain HTML, React, Vue, WordPress, Webflow — and its styles are fully isolated (Shadow DOM), so it can't clash with your site.

Quick start

Add the script once, then place the element wherever you want the game. That's a fully playable slot.

HTML
<!-- load the component from the CDN (auto-updates within v1) -->
<script src="https://cdn.jsdelivr.net/gh/dtekluva/neon2@v1/neon-slot.js"></script>

<neon-slot balance="1000" bet="25" currency="CRD"></neon-slot>
No build step, no npm, no dependencies. For the 3D engine add engine="3d" — Three.js loads itself from the same CDN automatically.

JavaScript API

Grab the element and call methods / set properties:

JS
const slot = document.querySelector('neon-slot');

slot.balance         // get / set the balance (number)
slot.bet             // get / set the current bet
slot.spinning        // read-only boolean

Server-authoritative outcomes

Use this for anything with real value. Without it, outcomes are decided in the browser (fine for demos, but a player could tamper with them).

Provide an async hook and the reels become "dumb" — they only animate to whatever your backend returns:

JS
slot.resolveSpin = async (bet) => {
  const res = await fetch('/api/spin', {
    method: 'POST',
    headers: { 'content-type': 'application/json' },
    body: JSON.stringify({ bet })
  }).then(r => r.json());

  // return one symbol index (0..5) per reel:
  return { reels: res.reels /*, win: res.win  (optional authoritative payout) */ };
};

// optional: make the RISK gamble server-authoritative too
slot.resolveRisk = async (phase, pot) => ({ won: await serverDecides() });

Always debit/credit balances on your server too — treat the component's balance as display only.

Configuration

Set the config property to customize symbols, payouts, and the bet→phase tiers:

JS
slot.config = {
  currency: 'CRD',
  reels: 4,
  symbols: [
    { glyph: '⚡', color: '#0ff7ff', value: 5,  weight: 5 },
    { glyph: '★', color: '#ffd84d', value: 12, weight: 3 },
    { glyph: '7', color: '#ff4d6d', value: 50, weight: 1 }
  ],
  payouts: { 4: 1.0, 3: 0.4, 2: 0.12 }, // win = bet * value * payouts[matchCount]
  betTiers: [                            // these are the bet buttons AND the phase ceiling
    { bet: 10,  maxPhase: 0 },          // bet 10 → only Phase 1
    { bet: 100, maxPhase: 3 }           // bet 100 → unlocks Bonus
  ],
  riskOdds: 0.5
};

Go live — initialize a player

The component is just the front-end. To play for real value, point it at the live game backend and pass the one required parameter: a userId — the player's id in your wallet system (Liberty hustleback). The backend uses it to read balances, debit bets, and credit wins.

Base URL: https://game.libertydraw.com/api  ·  You supply: userId (integer). Everything else (reels, win, balances) comes from the backend.

Wire the four hooks once and the whole loop is server-authoritative:

JS
const API     = 'https://game.libertydraw.com/api';
const USER_ID = '761';            // ← the player's hustleback id (you provide this)
const slot    = document.querySelector('neon-slot');
let roundId  = null;

const post = (path, body) =>
  fetch(API + path, { method:'POST', headers:{'content-type':'application/json'}, body:JSON.stringify(body) })
    .then(r => r.json());

// 1) SESSION — read the player's authoritative balances
const s = await post('/session', { userId: USER_ID });
slot.setBalance(s.balance);        // token (CRD) balance — bets are drawn from this
showWallet(s.walletBalance);       // naira wallet — where winnings are paid
showTotalWon(s.totalWonNaira);     // cumulative winnings (₦)

// 2) SPIN — server decides the reels and debits the bet (tokens)
slot.resolveSpin = async (bet) => {
  const r = await post('/spin', { userId: USER_ID, bet });
  roundId = r.roundId;
  if (r.balance != null) slot.setBalance(r.balance);   // token balance after the debit
  return { reels: r.reels, win: r.win };
};

// 3) RISK — server decides each 50/50 climb (optional gamble ladder)
slot.resolveRisk = async () => {
  const r = await post('/risk', { roundId });
  return { won: r.won };
};

// 4) BANK — server credits the win to the naira wallet
slot.addEventListener('bank', async () => {
  const r = await post('/bank', { roundId });
  showWallet(r.walletBalance);        // updated naira wallet
  showTotalWon(r.totalWonNaira);      // updated cumulative winnings
});
Where does userId come from? It's the player's account id in your backend — issue it server-side when the player opens the game. The component never invents it. (For a private integration you'd also pass a signed session token; ask us to enable X-Session-Token.)

API endpoints

All play endpoints are POST … application/json against https://game.libertydraw.com/api. Money timing: the bet is debited on /spin, the win is credited on /bank (the pot is held server-side through the RISK ladder; a bust pays nothing).

Method & pathBodyReturns
POST /session{ userId } balance (tokens), walletBalance (₦), totalWonTokens, totalWonNaira, tokensPerNaira, currency, nonce, serverSeedHash, config
POST /spin{ userId, bet } roundId, reels (4× index 0–5), count, symbol, win, pot, phase, maxPhase, balance (tokens), nonce
POST /risk{ roundId } won, phase, pot, atCap, status
POST /bank{ roundId } amount (tokens won), nairaCredited, walletBalance (₦), balance (tokens), totalWonTokens, totalWonNaira
GET /health—ok, mock_external
GET /config—symbols, payouts, bet tiers, phase multipliers
Bet tiers also set the phase ceiling: 10→Phase 1, 25→Phase 2, 50→Phase 3, 100→Bonus.

Tokens & payouts

A player has two balances, and the game moves between them:

BalanceFieldRole
Tokens (CRD)balanceWhat bets are debited from on every spin.
Wallet (₦)walletBalanceWhere winnings are paid when the player banks.
Conversion: 1 ₦ = 4 tokens. A win pot of N tokens pays N ÷ 4 naira to the wallet (returned as nairaCredited). So a 100-token pot banks as ₦25. The token balance is not increased by a win — winnings go to the wallet.

totalWonNaira is the player's cumulative banked winnings (₦), tracked by the game backend.

React

Custom elements work in React as-is. Use a ref for properties/events:

JSX
import { useEffect, useRef } from 'react';
// load once: <script src="https://cdn.jsdelivr.net/gh/dtekluva/neon2@v1/neon-slot.js"> in index.html

function Slot() {
  const ref = useRef();
  useEffect(() => {
    const el = ref.current;
    el.resolveSpin = async (bet) => api(bet);
    const onWin = (e) => addCoins(e.detail.amount);
    el.addEventListener('bank', onWin);
    return () => el.removeEventListener('bank', onWin);
  }, []);
  return <neon-slot ref={ref} balance="1000" bet="25" />;
}

Tell Vue it's a custom element, then bind normally:

FAQ

Will it clash with my site's CSS?

No. The component renders inside a Shadow DOM — its styles are sealed off from your page and vice-versa.

Does the 3D engine need extra setup?

No. With engine="3d" it loads Three.js from the same CDN automatically.

Can players cheat the outcomes?

Only if you rely on the built-in RNG. Wire resolveSpin to your backend and the browser can't fabricate wins — it can only request an animation.

Is it mobile-friendly?

Yes — it's responsive and fills its container. Set a max-width to taste.
